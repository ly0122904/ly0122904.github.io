Database Design and Development

The sample work is to illustrate essential relational database design modelling techniques, and shows how Oracle SQL can be used to implement a database. The practice included conceptual modelling using the crowâ€™s feet notation and the Unified Modelling Language (UML), and using SQL statements (DDL and DML) to build a database, then using queries in a variety of ways to help transform, populate and test.

The Software use Oracle 11g Express and SQL Developer, the existing databases were used as a base to re-build, modified and practice.

Showcase

This project takes an existing, populated database (Oracle 11g Northwind Database), re-design parts of it according to the provided requirements, implement those design changes, and then migrate all existing data from the original database to the newly-enhanced database. Part of the data migration will entail some data manipulation. then some test scripts was created to verify the accuracy of your data migration.

This project is to practice all the core DBAS course concepts, so will require analysis and design of an ERD, implementation of both Data Definition Language and Data Manipulation Language scripts, as well as using queries in a variety of ways to help transform, populate and test the validity of your work.

Entity Relationship Diagram

ERD

ER Diagrams illustrate the logical structure of databases. ER diagram deals with three major components of a database namely, Entity, Attributes and Relation.Each relationship in ER Diagram has a relationship between the two entities.Three types of relationship included: one to one, one to many and many to many. The relationship is established on the basis of primary keys and foreign keys.In this project, the original Northwind ERD and redesigned ERD are shown, there are 3 tables added.

Data Definition Language

DDL

Data Definition Language (DDL) script is a standard for commands that define the different structures in a database. DDL statements create, modify, and remove database objects such as tables, indexes, and users. Common DDL statements are CREATE, ALTER, and DROP

Data Manipulation Language

DML

DML statements are used to work with the data in tables.It is used to retrieve, store, modify, delete, insert and update data in database. Examples: SELECT, UPDATE, INSERT statements.
